# Help Content Source Of Truth

Обновлено: 2026-03-23

## Назначение
- Этот файл — **каноничный источник product-meaning copy** для in-app help screen FamilyRecipes.
- Если меняется смысловая рамка help / about в приложении, сначала обновляется этот файл, а потом уже app UI.

## Правило source of truth
- Нельзя считать help-контент обновлённым, если идея изменилась только в чате или только в app code.
- Актуальный цикл такой:
  1. Owner обсуждает новый смысл / framing с Codex.
  2. Codex обновляет `docs/help_content.md` и, если нужно, синхронизирует `docs/product_brief.md`, `docs/backlog.md`, `docs/decisions.md`, `docs/current_state.md`.
  3. Owner передаёт Claude задачу на sync help в приложении.
  4. Claude обновляет app-side help data и UI-экран.
  5. Claude фиксирует результат в `docs/sessions/` и `docs/current_state.md`.
  6. Codex проверяет, что `docs/help_content.md` и app UI не расходятся.

## App-side правило
- Help-контент не должен быть размазан строками по `HelpView`.
- После реализации в приложении должен быть один централизованный app-side источник help данных, например:
  - `apps/ios/FamilyRecipes/FamilyRecipes/FamilyRecipes/Features/Settings/HelpContent.swift`
- `HelpView` должен рендерить секции из этой структуры, а не хардкодить каждую фразу прямо во view.

## Что проверять при любом update
- Help screen всё ещё объясняет:
  - связь между людьми;
  - историю рецепта;
  - расстояние / эмиграцию / сепарацию;
  - повторяемость и путь к мастерству.
- Названия секций и смысл текста совпадают с app UI.
- Новые product insights сначала отражены здесь, а не только в коде.
