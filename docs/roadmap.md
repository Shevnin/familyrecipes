# Roadmap

Статус: обновлён 2026-02-25.

## R0 - Problem/Solution Fit [done]
Цель: выбрать один главный пользовательский сценарий и подтвердить ценность.

Результат:
- Сформирован `product_brief.md` — два режима (передатчик/мастер), двойное хранение рецепта.
- Выбран top flow: мама набирает рецепт → отправляет → сын видит оригинал + структурированную версию.

## R1 - Split Screen Prototype [current]
Цель: рабочий прототип top flow, доступный по URL.

Что сделано:
- `src/index.html` — split screen (два телефонных экрана: Передатчик + Мастер).
- Вкладка "Отправленные" у мамы.
- Деплой на GitHub Pages: https://shevnin.github.io/familyrecipes/src/index.html
- Эвристический парсер маминого текста → структурированная версия (ингредиенты + шаги).
- Запрос рецепта по ссылке: Мастер генерирует ссылку → родственник заполняет рецепт → рецепт в списке.
- Персистентность через localStorage (`recipe_requests`, `recipe_responses`).

Что дальше:
- Обратная связь от реального пользователя (мама).
- Итерация UI по результатам.

## R2 - First Delivery Loop
Цель: реализовать и проверить рабочий сценарий end-to-end.

Инициативы:
- Бэкенд или хранение (выход за пределы localStorage).
- Реальная связка мама ↔ сын (не на одной странице).
- Готовка по шагам на стороне сына.
