# Сессия 2026-02-24 — Mobile Link Submit Form

## Задачи
1. **request_link_generation** — таб "Запросить" на экране Мастера: форма создания запроса, генерация ссылки с UUID-токеном, localStorage, список запросов.
2. **mobile_link_submit_form** — при открытии `?r=<token>` показывается мобильная форма ответа на запрос рецепта.

## Что сделали
- Правый экран (Мастер): добавлены табы "Рецепты" / "Запросить".
- Таб "Запросить": форма (у кого + название блюда), генерация ссылки через `crypto.randomUUID()`, copy-to-clipboard, список запросов из localStorage.
- Reply flow (`?r=token`): при открытии ссылки скрываются оба телефона, показывается мобильная форма ответа.
  - Валидный токен → форма с предзаполненным названием блюда.
  - Невалидный токен → экран ошибки "Запрос не найден".
  - Уже выполненный запрос → "Рецепт уже отправлен".
  - Пустой текст рецепта → мягкая валидация.
- На submit: рецепт сохраняется в `recipe_responses`, статус запроса → `fulfilled`.
- Статусы в списке запросов: "ожидает" (оранжевый) / "выполнен" (зелёный).
- Адаптивная высота телефонов `min(812px, calc(100vh - 80px))`.

## localStorage-схема
- `recipe_requests`: `[{ id, token, requester, target_person, requested_title, status, created_at }]`
- `recipe_responses`: `[{ token, title, body, created_at }]`
