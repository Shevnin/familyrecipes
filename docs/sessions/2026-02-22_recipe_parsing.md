# Сессия 2026-02-22 — Парсинг рецепта

## Задача
P0 #4: заменить мок структурированной версии на реальный парсинг текста рецепта.

## Что сделали
- Удалён захардкоженный `MOCK_STRUCTURED`.
- Добавлена функция `parseRecipe(text)` — эвристический парсер на клиентском JS.
- Логика: строки с количествами/единицами (г, кг, шт, ст.л. и т.д.) или "по вкусу" → ингредиенты. Остальные строки → шаги.
- Fallback: если текст пустой или не удалось разобрать — показывается мягкое сообщение.

## Ограничения
- Парсинг эвристический, не LLM. Работает хорошо для структурированного ввода (ингредиенты с количествами на отдельных строках). Для потока сознания ("берешь картошку...") — всё уйдёт в шаги.
- Это прототип — цель проверить flow, а не точность парсинга.
