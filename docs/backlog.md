# Backlog (Discovery -> Delivery)

Формат:
- `P0` = критично для текущей фазы
- `P1` = важно, но после P0
- `P2` = позже

## P0
1. Сформулировать one-sentence product thesis.
2. Заполнить `idea canvas` по текущей гипотезе (`/Users/user/git-project/familyrecipes/docs/templates/idea_canvas.md`).
3. Утвердить приоритетный пользовательский flow.
4. Определить 3-5 критериев успеха для flow.
5. Разложить инициативы из flow в roadmap R0/R1.

## P1
1. Описать ключевые edge-cases выбранного flow.
2. Определить MVP boundary (что точно не делаем в первой сборке).
3. Подготовить первый набор task-card под delivery.

## P2
1. Определить подход к метрикам после запуска MVP.
2. Сформировать post-MVP гипотезы.
