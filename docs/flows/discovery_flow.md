# Discovery Flow (Idea -> Roadmap -> Backlog)

## Назначение
Это рабочий flow для наших продуктовых обсуждений до начала новой разработки.

## Шаги
1. **Idea framing**
   Что это за идея, для кого, какую боль закрываем.
2. **Problem validation**
   Почему это действительно проблема и как сейчас ее решают.
3. **Value proposition**
   Чем наше решение лучше/проще/быстрее альтернатив.
4. **User flow draft**
   Happy path, edge cases, ключевые точки отказа.
5. **Success criteria**
   Как поймем, что гипотеза работает (какие наблюдаемые сигналы).
6. **Initiatives**
   Какие продуктовые инициативы нужны, чтобы проверить и реализовать flow.
7. **Roadmap assembly**
   Расклад инициатив по релизам.
8. **Backlog prioritization**
   Детализация задач в формате приоритетов и зависимостей.

## Выходы цикла
- Обновленный roadmap.
- Обновленный backlog.
- Запись решения в decision log.
