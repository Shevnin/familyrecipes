# Core Flow v1 — Link → View → Save → My → Share

## Цель
Пользователь получает рецепт по ссылке, быстро понимает “о чём рецепт”, сохраняет его, находит в “Мои рецепты” и может поделиться ссылкой.

## Шаги флоу
1. User открывает `recipe.html?id=...`
2. Видит страницу рецепта (название, время/порции, ингредиенты, шаги)
3. Нажимает **Сохранить**
4. Переходит в **Мои рецепты** (`my.html`) и видит рецепт в списке
5. Нажимает **Поделиться** (копирование ссылки / системный share позже)

## Экраны и элементы

### 1) Recipe (по ссылке)
Состав:
- Заголовок рецепта
- Мета: время, порции (если есть)
- Ингредиенты (список)
- Шаги (список)
- CTA: **Сохранить**
- CTA: **Поделиться**
- Toast/feedback для save/share

Тексты v1:
- Save toast: `Сохранено`
- Share toast success: `Ссылка скопирована`
- Share toast fail: `Не удалось скопировать`

### 2) My recipes
Состав:
- Заголовок “Мои рецепты”
- Список сохранённых (каждый — ссылка на рецепт)
- Empty state

Тексты v1:
- Empty: `Пока ничего не сохранено. Открой рецепт и нажми “Сохранить”.`

Опционально v1.1:
- Удаление из сохранённых (кнопка/иконка на элементе)

## Acceptance Criteria (v1 прототип)
- Открытие `src/recipe.html?id=1` рендерит рецепт из мок-данных.
- Нажатие “Сохранить” добавляет `id` в localStorage и показывает `Сохранено`.
- `src/my.html` читает сохранённые `id` и отображает список ссылок на рецепты.
- Если сохранённых нет — показывается empty state.
- “Поделиться” копирует текущий URL в буфер (если доступно) и показывает корректный toast.
- Повторное сохранение того же рецепта не создаёт дубликаты в “Мои рецепты”.

## Не делаем в v1 (явно)
- Аккаунты/логин
- Облако/синхронизация
- Публичные профили
- SEO/фид/поиск по всему каталогу
- Полноценный share sheet (пока копирование ссылки)
