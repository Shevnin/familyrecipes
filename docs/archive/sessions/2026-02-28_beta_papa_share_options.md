# Сессия: Бета-сценарий "папа сразу хочет поделиться рецептом"

Дата: 2026-02-28

## Контекст
Представили сценарий: пользователь устанавливает бету и сразу хочет не запрашивать, а поделиться рецептом.

## Продуктовые варианты

1. **Share-first entrypoint (recommended for beta)**
- На первом экране два равноправных CTA:
  - `Запросить рецепт`
  - `Поделиться рецептом`
- `Поделиться рецептом` ведёт в простой flow: ввести рецепт -> поделиться ссылкой/текстом.
- Плюсы: быстрый "aha moment", меньше трения.
- Риск: усложнение IA (два главных пути вместо одного).

2. **Быстрая ссылка для донора**
- Из режима `Поделиться` есть вариант: `Отправить ссылку тому, кто продиктует`.
- Подходит для родственников без приложения.
- Плюсы: покрывает mixed-tech семью.
- Риск: больше ветвлений и UX-решений по статусам.

3. **Семейный inbox отправок**
- Пользователь отправляет "рецепт в семью", получатели принимают/сохраняют.
- Плюсы: сильная social-механика.
- Риск: заметно выше сложность backend и прав доступа.

## Рекомендация
Для текущей беты: взять **вариант 1 + элемент варианта 2**.
- В MVP оставить единый экран "Поделиться":
  - `Сам заполню`
  - `Отправить ссылку`

## Статус
Это продуктовая заметка, не финальное архитектурное решение.  
Если подтверждаем, оформляем как новые задачи в backlog (`IOS/ANDROID`) и parity-спеку.
